# 要件定義: 教育データ発見エンジン「見えないつながりを見えるようにする」

**作成日**: 2026-04-05
**ステータス**: ドラフト（レビュー待ち）
**基盤リサーチ**: 10本のディープリサーチ（KDD・疫学・心理測定学・ネットワーク科学・行動経済学・システム思考・動機付け科学・発達心理学・教育社会学・エスノメソドロジー・パターンランゲージ・批判的データ研究・情報理論・複雑系科学 等）

---

## 1. ビジョンと目的

### 1.1 ミッション

> **教科・テスト・時間の壁を超えて、教育データに隠れた「つながり」を発見し、
> 教員が理解・活用できる形で可視化するシステム**

### 1.2 コアコンセプト: 3種類の「見えない」

| 種類 | 説明 | 例 |
|------|------|-----|
| **データ的に見えない** | 単純集計・相関では検出不能な非線形・文脈依存的関係 | 相互情報量でのみ検出される教科間依存 |
| **認知的に見えない** | データは示しているが、人間のバイアスが認識を妨げる | 確証バイアスで見逃す反例パターン |
| **構造的に見えない** | データの集約・平均化が隠す個別の複雑性 | 成績平均というブラックボックスの中の設問レベル構造 |

### 1.3 根本原則

1. **ネットワークが中心メタファー**: すべてのつながりを「ノードとエッジ」で表現・可視化する
2. **層で段階的に深める**: 表層（記述）→ 中層（パターン）→ 深層（構造・因果）
3. **パターンランゲージで橋渡し**: 発見を「コンテクスト-問題-解決」形式で命名し、教員の共通言語にする
4. **ブラックボックスを開く**: 成績平均・偏差値の背後にある設問レベルの構造を常に参照可能にする
5. **測定していないものを自覚する**: 批判的データ研究の視点で、分析の限界を明示する

## 2. スコープと制約

### 2.1 対象データ

| データ種別 | 粒度 | 期間 | 推定レコード数 |
|-----------|------|------|--------------|
| 設問単位テスト | 正誤 × 選択肢（全教科） | 各テスト | 500-1000人 × 30-50問 × 年間10-20回 |
| 成績 | 観点別3次元 × 教科 × 学期 | 3年間 | 500-1000人 × 10教科 × 6学期 |
| 出欠 | 日単位 | 3年間 | 500-1000人 × 約600日 |
| 模試 | 教科別偏差値 + 志望校判定 | 年3-5回 | 500-1000人 × 年3-5回 |
| 入試 | 教科別点数 + 入試区分 | 入学時 | 各学年分 |
| アンケート | 構造化（リッカート） + 自由記述 | 年1-2回 | 500-1000人 |

### 2.2 対象外（Phase 1）

- リアルタイムLMSログ（GIGAスクール環境の成熟待ち）
- マルチモーダルデータ（動画・生体データ等）
- 学校外データ（塾・家庭環境の直接計測）

### 2.3 技術的制約

- Google Drive上での重い処理は不適。分析はローカル/クラウド環境で実行
- Git管理下に置く（Google Drive外のリポジトリ推奨）
- 人事異動を前提としたハンドオフ可能な設計

### 2.4 法的制約

- 個人情報保護法（APPI）準拠必須
- 文科省「教育データの利活用に係る留意事項」第3版（2025年3月）準拠
- 仮名加工情報として処理（内部分析用途）
- 報告・共有時はk-匿名性（k≥5）を保証

## 3. 発見対象の定義 — 5層モデル

### Layer 1: 設問間のつながり（ミクロ層）

**FR-1.1 設問ネットワーク分析**
- 全教科の設問データからIsingモデル（二値ネットワーク心理測定）を推定
- 一般能力を統制した後の設問間の直接依存関係を可視化
- Bridge Centrality計算により教科間をつなぐ「橋渡し設問」を特定
- ツール: bootnet + qgraph (R)

**FR-1.2 前提条件ネットワーク**
- ベイジアンネットワーク構造学習により、スキル間の前提関係を自動発見
- Knowledge Space Theoryとの統合（既存ドキュメント Section 3.2）
- 教科横断の前提チェーン検出（例: 数学→理科→技術）
- ツール: bnlearn (R) / pgmpy (Python)

**FR-1.3 誤答パターン分析**
- CDM（DINA/G-DINA）による認知診断プロファイル生成
- Q-matrix: 教員との協働構築（各教科5-8属性、各属性3問以上）
- Tatsuoka Rule Space Method による認知バグ分類
- 誤答選択肢の診断的活用（各選択肢が特定の誤概念に対応）
- 現象学的記述の視点: 異なる誤答＝質的に異なる理解のカテゴリー
- ツール: GDINA (R), CDM (R)

**FR-1.4 コミュニティ検出**
- 設問の共正答ネットワーク上でLouvain法によるクラスタリング
- 発見されたコミュニティとカリキュラム上の教科区分の不一致を検出
- 教科境界を超えたスキルクラスタの発見
- ツール: igraph (R/Python)

### Layer 2: 生徒タイプのつながり（メゾ層）

**FR-2.1 潜在クラス分析**
- LCAにより質的に異なる生徒タイプ（3-6クラス）を発見
- 診断プロファイル（CDM出力）をLCA入力として使用
- BIC/AICによる最適クラス数選択
- 各クラスの特徴プロファイル（項目応答確率）を可視化
- ツール: poLCA (R)

**FR-2.2 軌跡モデリング**
- GBTM（Group-Based Trajectory Modeling）で離散的な成績軌跡群を発見
- 「一貫高位」「後半上昇」「2年急落」「低位安定」等のパターン
- 軌跡群メンバーシップの予測因子分析
- ツール: lcmm (R)

**FR-2.3 状態遷移分析**
- LTA（Latent Transition Analysis）で学期間の状態遷移確率を推定
- どの遷移が一般的で、どの遷移が稀か
- 遷移を予測する共変量の特定（介入ポイントの発見）
- ツール: LMest (R)

**FR-2.4 「やらない」vs「できない」診断**
- 動機付け科学に基づく7指標の診断テーブル
  1. 簡単な問題の正答率（高→動機的、低→スキル的）
  2. テスト前半 vs 後半の正答率変化
  3. 高ステークス vs 低ステークスの成績差
  4. 無回答率（「放棄」の代理指標）
  5. 出欠パターン（選択的欠席→動機的）
  6. アンケート自己効力感スコア
  7. 教科間の成績分散（高分散→動機的選択性）

### Layer 3: 教科間のつながり（マクロ層）

**FR-3.1 教科間偏相関ネットワーク**
- 教科単位の成績データから偏相関ネットワークを推定
- 一般学力因子を除去した後に残る教科間の直接関係
- Bridge Centralityで「万能ボトルネック教科/スキル」を特定
- ツール: bootnet + qgraph (R)

**FR-3.2 アソシエーションルールマイニング**
- Aprioriアルゴリズムで教科横断の設問レベルルールを発見
- 閾値: support ≥ 0.05, confidence ≥ 0.50, lift > 1.2
- 多重比較補正: FDR制御（q=0.10）を適用
- 特に注目: 教科をまたぐルール（例: {数学_誤答A, 理科_誤答B} ⇒ {英語_誤答C}）
- ツール: arules (R) / mlxtend (Python)

**FR-3.3 正準相関分析 (CCA)**
- 教科ペア間の共有潜在スキル次元を発見
- 例: 「数学×物理」の第1正準変量は代数操作の共有、第2は論理推論の共有
- ツール: cancor() (R) / sklearn CCA (Python)

**FR-3.4 時間的先行関係分析**
- RI-CLPM（Random Intercept Cross-Lagged Panel Model）で教科間の時間的因果を推定
- 例: 英語読解→社会科（先行）か、社会科→英語読解か？
- 要件: 3時点以上の縦断データ
- ツール: lavaan (R)

**FR-3.5 相互情報量分析**
- Pearson相関では検出不能な非線形依存関係の検出
- 相互情報量が高いのに相関が低い変数ペア＝非線形つながりの存在
- JMIM（Joint Mutual Information Maximization）による変数選択
- ツール: sklearn mutual_info (Python)

### Layer 4: 時間のつながり（時系列層）

**FR-4.1 成長曲線モデリング**
- LGM（Latent Growth Model）で個人別の成長率を推定
- 入学時は同等でも成長率が異なる生徒群の発見
- 成長率を予測する変数（出欠、入試成績、アンケート等）の特定
- ツール: lavaan (R)

**FR-4.2 シーケンシャルパターンマイニング**
- PrefixSpanで正答/誤答の時間的順序パターンを発見
- 「ある単元の習得 → 別の単元の習得」の最適順序の発見
- ツール: SPMF (Java) / mlxtend (Python)

**FR-4.3 ティッピングポイント検出**
- 複雑系科学に基づく臨界点の検出
- 出欠率の区分回帰で閾値を発見（例: 出席率80%で成績急落）
- 早期警告シグナル: 分散の増大が臨界遷移に先行する
- ツール: R strucchange / Python ruptures

**FR-4.4 スペーシング効果の検出**
- 「詰め込みスコア」: 定期テスト成績 - 模試成績の差
- 差が大きい生徒＝短期記憶依存（スペーシング不足）
- 教科別のスペーシング効果差異の分析

### Layer 5: メタ分析（データ自体の構造）

**FR-5.1 テスト品質分析**
- CTT: 全設問の難易度指数(p)・識別力指数(D)・錯乱肢分析
- IRT: Rasch（単元テスト）、2PL（定期テスト）、3PL（入試・模試）
- 品質フラグ: D < 0.20 または p < 0.10 / p > 0.90 の設問を自動検出
- ツール: jMetrik (無料) / mirt (R)

**FR-5.2 公平性分析**
- DIF（Differential Item Functioning）: コース別・性別での設問機能差異検出
- Mantel-Haenszel法（N≥200/群）
- 大きなDIFを示す設問 → 教員にレビュー依頼
- ツール: mirt / difR (R)

**FR-5.3 偏差値の妥当性検証**
- 分布形状チェック（二峰性検定、天井/床効果）
- 二峰分布の場合は偏差値使用不可フラグ
- IRT等化による年度間比較（共通アンカー項目必要）

**FR-5.4 教員評価バイアス検出**
- マルチレベルモデルで「同じ能力の生徒がクラスによって異なる評価を受ける」度合いを推定
- ICC（級内相関）の計算
- 観点別評価3次元間の整合性チェック

## 4. 非機能要件

### 4.1 倫理・プライバシー

**NFR-E1**: 全データは仮名加工情報として処理。生徒IDは分析用ハッシュに置換
**NFR-E2**: 報告・可視化時にセルサイズ < 5 の場合は自動抑制
**NFR-E3**: 「リスク生徒」等のラベリングは行わない。代わりに「サポート検討候補」と表現
**NFR-E4**: 予測モデルの使用制限:
  - 個人に対する不可逆的決定（クラス分け・コース配置）への直接使用禁止
  - 予測なしでも正当化できる介入のみ、予測を根拠にしてよい
  - n < 30 のサブグループへの予測適用禁止
**NFR-E5**: 公平性監査: 全モデルの性能指標をコース別・性別で分解報告
**NFR-E6**: 年1回以上のデータ取扱い見直し（MEXT自己点検に準拠）

### 4.2 可解釈性・教員インターフェース

**NFR-I1**: 全ての発見は「コンテクスト-問題-解決」のパターンランゲージ形式で命名
**NFR-I2**: ネットワーク可視化は教科別色分け + エッジ太さ＝強度 + ブリッジノード強調
**NFR-I3**: XAI（説明可能AI）: 予測モデル使用時はSHAP値による個別説明を付与
**NFR-I4**: Progressive Disclosure: 概要（信号灯）→ パターン一覧 → 詳細ネットワーク → 生データ
**NFR-I5**: 自然言語アノテーション: 「三角関数の習得は波動物理の成功率を3倍に高めます」

### 4.3 再現性・持続性

**NFR-R1**: 全分析コードはGit管理。コミットメッセージに分析目的を記述
**NFR-R2**: データバージョニング（DVC or 手動スナップショット）
**NFR-R3**: 各分析はJupyter/R Markdownで再実行可能な形式
**NFR-R4**: データ辞書: 全変数名・データソース・意味を一元管理
**NFR-R5**: 人事異動対応: 年度末にハンドオフドキュメント更新

### 4.4 パフォーマンス

**NFR-P1**: バッチ分析（日次/週次）で十分。リアルタイム性は不要
**NFR-P2**: 全分析パイプラインが1時間以内に完了すること（N=1000基準）
**NFR-P3**: ダッシュボードの初期ロードは5秒以内

### 5.2 発見インターフェースの4画面

**画面1: つながりマップ**
- 設問レベル/教科レベルの偏相関ネットワーク
- ノード: 設問カテゴリ or 教科（色分け）
- エッジ: 偏相関の強さ（太さ）と方向（あれば矢印）
- Bridge Centralityが高いノードをハイライト
- フィルタ: 教科選択、閾値スライダー、学年選択
- インタラクション: ノードクリック → 関連する発見パターン表示

**画面2: パターンカタログ**
- 発見されたパターンのカード型一覧
- 各パターンは「コンテクスト-問題-解決」のパターンランゲージ形式
- 例:
  ```
  パターン名: 「つまずきの連鎖」
  コンテクスト: 数学の二次関数で誤答Bを選択する生徒群
  問題: この生徒群は物理の放物運動でも同種の誤りを犯す（lift=2.3）
  解決: 共通の前提スキル（符号付き座標操作）の補強が両教科に波及する
  信頼度: confidence=0.78, support=0.12
  ```
- ソート: 信頼度、影響範囲（関連教科数）、新規性
- タグ: #教科横断 #誤概念 #時間パターン #動機付け #ボトルネック

**画面3: 個別診断プロファイル**
- 生徒選択 → CDMによるスキルマスタリーマップ
- 教科ごとの属性マスタリー（バイナリベクトル）を可視化
- 所属する潜在クラス（LCA結果）の表示
- 軌跡群（GBTM結果）での位置
- 「やらない」vs「できない」診断スコア
- **注意**: NFR-E3に基づき、ラベリングではなく「特徴の記述」として表示

**画面4: テスト品質レポート**
- 各テストの項目分析サマリー（難易度・識別力分布）
- 品質問題フラグ付き設問の一覧
- DIF検出結果（コース別・性別）
- 年度間のIRT等化結果（共通アンカー項目がある場合）

## 6. Q-matrix協働構築プロセス（最重要依存タスク）

CDM、Knowledge Graph、診断プロファイルの全てがQ-matrixに依存するため、
これは技術開発と並行して進める最優先の協働タスクである。

### 6.1 プロセス

```
Phase 1: 教科主任との初期ワークショップ（各教科2時間）
├── 教科の主要スキル/属性の洗い出し（5-8個/教科）
├── 既存テストの設問と属性の対応付け
└── 前提関係の仮説的マッピング

Phase 2: データによる検証・修正
├── GDINA R パッケージによるQ-matrix検証
├── 統計的に問題のある対応の修正提案
└── 教員との再協議

Phase 3: 運用・更新
├── 新テスト作成時にQ-matrixエントリを同時記入
├── 年度末にQ-matrixの有効性レビュー
└── 属性定義の教科間整合性チェック
```

### 6.2 対象教科（優先順位）

| 優先度 | 教科 | 理由 |
|--------|------|------|
| 最優先 | 数学 | 前提関係が最も明確、既存研究が豊富、他教科への波及効果大 |
| 最優先 | 理科（物理・化学） | 数学との教科横断が最大 |
| 高 | 英語 | 読解力が全教科に波及（ボトルネック仮説検証） |
| 中 | 国語 | 技術的読解力の教科横断効果 |
| 中 | 社会 | 英語・国語との関連検証 |

## 7. システム思考に基づく発見フレームワーク

リサーチで得られたシステムアーキタイプを、教育データから検出すべきパターンとして組み込む。

### 7.1 検出すべきシステムアーキタイプ

| アーキタイプ | 教育での現れ方 | データシグナル |
|-------------|---------------|--------------|
| **成功が成功を呼ぶ** | 良い成績→自信→さらに良い成績 / 悪い成績→自信喪失→さらに悪化 | 成績の分散が時間とともに拡大（ジニ係数の上昇） |
| **応急処置の失敗** | テスト前の詰め込み→短期的成績回復→長期的理解不足 | 定期テスト好成績だが模試で急落（スペーシング効果の欠如） |
| **負荷の転嫁** | 塾への依存→学校の学びへの受動性 | 学校成績より模試成績が高い群の存在 |
| **成長の限界** | ある教科の成績向上が頭打ち→ボトルネックスキルの制約 | 成長曲線の漸近線 + ネットワーク分析でボトルネック特定 |

### 7.2 フィードバックループの検出

- 強化ループ: 出欠悪化→成績低下→動機低下→出欠さらに悪化
  - 検出: Cross-Lagged Panel で出欠→成績→出欠の循環的因果
- 平衡ループ: 成績改善→安心→努力低下→成績低下→焦り→努力増加
  - 検出: 成績の周期的振動パターン（自己相関分析）

## 9. 成功基準

### 定量基準

| 指標 | 目標 |
|------|------|
| 教科横断つながりの発見数 | Phase 1で10件以上 |
| 教員が「知らなかった」と評価する発見 | 発見の30%以上 |
| Q-matrix構築完了教科数 | Phase 2までに3教科 |
| ダッシュボード月間アクセス数 | 教員の50%が月1回以上 |
| パターンカタログのパターン数 | Phase 3までに30件以上 |

### 定性基準

- 教員が授業研究の中で発見パターンを引用する
- 「つまずきの連鎖」等の命名が教員間の共通語になる
- データに基づく介入提案が校内で議論される
- 新テスト作成時にQ-matrixが参照される

### アンチパターン（避けるべき失敗）

| 失敗パターン | 回避策 |
|-------------|--------|
| 技術的に正しいが教員に伝わらない | パターンランゲージ + Progressive Disclosure |
| 発見が多すぎて何が重要かわからない | Bridge Centrality + lift > 1.5 でフィルタ |
| データが教員の統制手段と認識される | 発見は「支援のため」の枠組みで共有 + NFR-E3 |
| 人事異動で継続不能 | NFR-R1-R5 の再現性要件 |
| 過剰なラベリングが自己成就予言を生む | NFR-E3 + NFR-E4 |
