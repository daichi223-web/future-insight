### 3. 多角的分析（3ラウンド）

**第1ラウンド: 「何を見つけるか」— 発見対象の分類学**
- 5層モデルを導出: 設問間(ミクロ) → 生徒タイプ(メゾ) → 教科間(マクロ) → 時間(時系列) → メタ(データ自体)
- 各層に最適な手法を対応付け
- 核心的洞察: 教科は人為的区分であり、認知的スキルの真のネットワーク構造はそれと異なる

**第2ラウンド: 「誰のために・何のために」— 矛盾と優先順位**
- 3つのテンション特定: 深さ vs 理解しやすさ、網羅性 vs 実現可能性、自動発見 vs 暗黙知
- 優先順位マトリクス: Network Psychometrics, Association Rules, LCA, Community Detection が「最優先」
- パターンランゲージがテンション1を解消する鍵

**第3ラウンド: 「どう統合するか」— システム設計**
- 3層アーキテクチャ: データ基盤 → 発見エンジン → 発見インターフェース
- 4画面設計: つながりマップ、パターンカタログ、個別診断、テスト品質

## 判断ログ

### Network Psychometrics を最重要手法とした理由

複数の独立したリサーチエージェント（#6 設問レベル分析、#7 ネットワーク手法）が独立に同じ結論に到達:
- **偏相関**により一般能力因子を除去した「真のつながり」が見える
- Ising model は二値データ（正誤）に直接適用可能
- Bridge Centrality で教科横断ボトルネックを自動特定
- N=500-1000 で十分な推定精度（2025年シミュレーション研究で確認）
- 教員への可視化が直感的（ネットワーク図）

### パターンランゲージを通信層とした理由

社会学リサーチ（#10）で、井庭崇（慶應）の16年間1,610パターンの実績を発見。「コンテクスト-問題-解決」形式は:
- 統計的発見を教員の行動可能な知識に変換する
- 授業研究の文脈に自然に統合できる
- 命名されたパターンが教員間の共通語になる

### 「やらない」vs「できない」診断を組み込んだ理由

動機付け科学リサーチ（#9）が示した核心的洞察: スキル分析だけでは根本原因を見逃す場合がある。同じ「低成績」でも動機的原因とスキル的原因では介入が全く異なる。7つの指標が既存データから計算可能。

### CDMのQ-matrixを最重要依存タスクとした理由

CDM、Knowledge Graph、診断プロファイルの全てがQ-matrixに依存する。技術的に最も洗練された手法でも、Q-matrixなしでは動かない。教員との協働構築に時間がかかるため、Phase 0から並行開始が必要。

### TDA（位相データ分析）を見送った理由

ネットワークリサーチ（#7）の結論: 理論的に興味深いが、教育分野での先行事例がほぼなく、教員への解釈説明が極めて困難。Phase 4以降の探索的試行としてのみ位置づけ。

### 倫理要件でラベリング禁止とした理由

社会学リサーチ（#10）のラベリング理論 + 倫理リサーチ（#1）の知見: 「リスク生徒」というラベルは自己成就予言を生み、教育的に有害。「サポート検討候補」等の非ラベリング表現を徹底。

### 記録すべき教訓
- **compute投入は惜しまない**: 10本の並列リサーチで広範な知見を効率的に取得できた
- **独立したエージェントが同じ結論に収束**: Network Psychometrics が最重要手法であることを複数エージェントが独立に結論。信頼性の高い判断
